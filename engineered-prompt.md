## git branching strategy 
once a feature passes unit test, merge them into dev and delete the feature branch. use Dev branch for developing ideas and test branch for testing new appraches to design decisions. Use feature branches named after the feature to test implementation of new features. Once a feature is merged into dev, run end-to-end test. once testing coverage is 80% or higher merge into main
